Pow is a zero-configuration Rack server for Mac OS X. It makes developing Rails and Rack applications stupid easy. You can install it in ten seconds from the terminal (seriously!) and have your first app up and running in under a minute. No mucking around with your /etc/hosts file, no compiling Apache modules, no editing configuration files or installing preference panes. And running multiple apps is cake.

How does it work? A few simple conventions eliminate the need for tedious configuration. Pow runs as your user on an unprivileged port, and includes both an HTTP and a DNS server. The installation process sets up a firewall rule to forward incoming requests on port 80 to Pow. It also sets up a system hook so that all DNS queries for *.test resolve to your local machine.

To serve a Rack app, just symlink it into your ~/.pow directory. The name of the symlink is the hostname you use to access the app. So if you symlink /Volumes/37signals/basecamp into ~/.pow/basecamp, it'll handle all HTTP requests for *.basecamp.test.
